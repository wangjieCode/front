# 2026-02-10

## 背景

- 修复 monorepo 架构评审中发现的问题：无效脚本、构建语义错误、Turbo 配置不规范、缓存产物声明错误、多锁文件并存。

## 变更

- `backend/package.json`
  - `build` 改为 `node --import tsx scripts/build.ts`（运行时模式）。
  - `start` 改为 `node --import tsx src/index.ts`。
  - `start:worker` 改为 `node --import tsx src/worker.ts`。
  - `tsx` 调整为运行时依赖，支持生产环境 `--prod` 安装后直接启动。
  - 删除已不存在脚本文件对应的 scripts 项。
  - 补充现存脚本入口：`test:path-utils`、`test:new-prompt`。
- `backend/scripts/build.ts`
  - 新增 tsx 构建入口脚本，用于 monorepo 构建链路统一。
- `turbo.json`
  - 将 `pipeline` 更正为 `tasks`。
  - `build.outputs` 调整为 `dist/**`、`build/**`、`backend/public/**`。
  - `dev` 任务补充 `persistent: true`。
- `package.json`（根）
  - 增加 `packageManager: pnpm@10.14.0`，修复 Turbo 2.x 的 workspace 解析失败。
- `packages/shared/package.json`
  - `types` 从 `./src/index.ts` 调整为 `./dist/index.d.ts`。
  - `exports["."].types` 置于 `import/require` 之前，消除 `types condition will never be used` 告警。
- `frontend/package.json`
  - 增加 `"type": "module"`，消除 Vite CJS Node API 弃用告警。
- `deploy.mjs`
  - 部署前构建改为工作区级 `pnpm install --frozen-lockfile` + `pnpm turbo run build`（带 filter）。
  - 打包内容改为 monorepo 运行必需集：根 `package.json`/`pnpm-lock.yaml`/`pnpm-workspace.yaml`、`backend/src`、`backend/public`、`packages/shared/dist` 等。
  - 远端安装改为在工作区根执行 `pnpm install --prod --frozen-lockfile --filter @front/shared --filter web-frontend-intern-assistant-backend`，兼容 `workspace:*` 与 `tsx` 运行依赖。
- 锁文件治理
  - 删除 `backend/pnpm-lock.yaml`、`frontend/pnpm-lock.yaml`。
  - 删除根目录 `package-lock.json`，统一 pnpm。
  - `.gitignore` 增加对子包 `pnpm-lock.yaml` 的忽略。
- 文档
  - 新增规格：`docs/specs/spec-monorepo-workspace.md`。
  - 更新规格索引与技术架构说明。

## 影响

- 根 `turbo build/test` 语义恢复为“执行并结束”的任务，不再被后端启动逻辑阻塞。
- 依赖锁定策略统一到根锁文件，减少工作区依赖解析偏差。

## 验证

- 已执行：
  - `pnpm install`：成功（用户本地执行）。
  - `pnpm -w build`：Turbo 工作区已可解析；失败点为 backend 现存 TypeScript 类型错误（与本次 monorepo 架构修复无直接耦合）。
  - `pnpm -w test`：Turbo 工作区已可解析；失败点为 frontend Jest 环境缺少 `TextEncoder`（测试环境配置问题）。
  - `pnpm --dir packages/shared dev`：已无 `types condition` 警告。
  - `pnpm --dir frontend build`：已无 Vite CJS Node API 弃用警告，构建成功。
  - `node --check deploy.mjs`：语法检查通过。

## 风险

- `pnpm-lock.yaml` 需要在一次完整安装后刷新，避免历史锁文件残留导致 CI 偏差。
