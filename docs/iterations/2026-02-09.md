### 影响

- 旧字段残留不会再触发前端登录态误判。

### 验证

### 风险

- 变更后用户需重新登录一次以写入新字段。

## 补充迭代（老账号密码初始化）

### 背景

- 已存在但无密码哈希的历史账号，调用 `POST /api/auth/login` 时不会写入密码，导致响应 `hasPassword=false`。

### 变更

- 调整后端登录逻辑：若账号存在且 `passwordHash` 为空，强制校验本次提交密码并写入 `password_hash`。
- 登录成功响应中的 `hasPassword` 改为基于本次流程结果计算，确保初始化后立即返回 `true`。
- 同步更新认证规格文档，补充该场景的业务规则与验收标准。

### 影响

- 历史无密码账号首次登录会完成密码初始化，后续登录走密码校验。
- 消除“登录成功但未设置密码”的状态不一致问题。

### 验证

- 代码路径确认：`existing user + empty passwordHash` 分支现在会写入 `password_hash`。
- 接口行为预期：首次初始化后响应 `hasPassword=true`。

### 风险

- 历史账号首次初始化密码后，必须使用该密码登录，忘记密码需通过改密流程处理。

## 补充迭代（鉴权切换为 JWT）

### 背景

- 现有鉴权依赖 `x-user-id` / `x-username` 自定义头，不符合标准 JWT 鉴权实践。

### 变更

- 新增后端 JWT 工具，登录成功后签发 token，并在响应中返回 `token`。
- `requireAuth` 中间件改为校验 `Authorization: Bearer <jwt>`，并通过 `sub` 关联用户。
- `/api/auth/verify`、`/api/auth/users/:id/password`、`/api/auth/users/:id/username` 改为统一复用 `requireAuth`。
- 前端 `authUtils.getAuthHeaders()` 改为注入 Bearer token，移除 `x-user-id`/`x-username` 透传。
- 移动端登录补齐密码输入并统一走 JWT 登录返回。
- 同步更新认证规格与技术文档。

### 影响

- 所有受保护 API 现在只接受 Bearer JWT。
- 旧的自定义用户头鉴权路径被移除。

### 验证

- 代码检索确认前端业务请求不再设置 `x-user-id`/`x-username`。
- 登录成功链路包含 `token` 返回并写入 `fi_auth_token_v2`。

### 风险

- 未配置 `JWT_SECRET` 时将使用开发默认值；生产环境需显式配置强随机密钥。

## 补充迭代（JWT 改为社区标准库实现）

### 背景

- 认证链路虽使用 JWT 格式，但签发/验签逻辑为手写实现，不符合“使用社区标准方案”的要求。

### 变更

- 后端新增依赖：`jsonwebtoken`、`@types/jsonwebtoken`。
- 重写 `backend/src/utils/jwt.ts`：改为使用 `jsonwebtoken` 进行 JWT 签发与校验。
- 保持现有接口契约不变：token 结构、默认有效期（7 天）、Bearer 提取方式保持一致。
- 同步更新认证规格与技术文档。

### 影响

- JWT 逻辑从自研实现切换为社区标准库实现，维护性与兼容性更好。

### 验证

- 代码路径确认 `signAuthToken`/`verifyAuthToken` 已切换到 `jsonwebtoken`。
- 认证中间件与登录接口无需改动调用方式。

### 风险

- 依赖新增后，部署环境需保证安装后端新依赖。

## 补充迭代（登录后刷新项目列表）

### 背景

- 用户在未登录状态进入项目页后，登录成功不会自动重新请求项目列表接口，页面可能停留在旧状态。

### 变更

- 登录成功时在前端广播统一事件 `fi:login-success`。
- `ProjectsPage` 与 `MobileProjectsPage` 监听该事件并自动调用项目列表加载。
- `ProjectSelector` 与 `MobileProjectSelector` 同步监听该事件，登录后自动刷新可选项目数据。
- 同步更新项目管理规格文档。

### 影响

- 登录成功后，无需手动刷新页面即可看到最新项目列表。

### 验证

- 在未登录时进入项目页，完成登录后项目列表会自动请求并更新。
- 项目选择器在登录成功后会自动刷新数据源。

### 风险

- 事件驱动刷新依赖浏览器全局事件，后续若拆分微前端需统一事件命名约定。
