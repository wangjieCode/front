# 迭代记录：2026-02-09

## 背景

- 平台需要提供 Codex 可用的 GPT 系列模型，同时保留既有 IFLOW 模型选择能力。

## 变更

- 前端模型选项改为双集合并：保留 IFLOW 模型，并新增 Codex GPT 系列（`codex/gpt-5-codex`、`codex/gpt-5`、`codex/gpt-5-mini`、`codex/gpt-4.1`、`codex/gpt-4.1-mini`）。
- 后端模型白名单同步改为双集合并，支持 `iflow/*` 与 `codex/*`。
- 默认模型统一调整为 `codex/gpt-5-codex`。
- 更新模型选择规格文档与 SDK 验证脚本规格文档，默认模型与支持列表同步为双模型体系。
- 新增后端模型配置接口 `GET /api/conversations/models`，返回默认模型与每个模型的可用状态（`enabled`）。
- 启动阶段新增 Codex Auth 探测：调用 `codex/gpt-5-codex` 最小请求验证可用性；失败时自动禁用全部 `codex/*` 模型选项。
- 前端模型下拉切换为读取后端配置，`enabled=false` 的模型在下拉中置灰禁选。
- 新增后端单元测试 `modelAvailabilityService.test.ts`，覆盖 Codex Auth 探测成功/失败下的模型启用状态与默认模型回退逻辑。
- 修复“Codex 会话卡在 AI 思考中”问题：消息发送阶段统一解析并强制传递可用模型（优先请求模型，否则会话模型，不可用自动回退默认可用模型）。
- 为 Neovate SDK 执行增加统一超时控制（`NEOVATE_EXEC_TIMEOUT_MS`，默认 180000ms），超时后中断并返回错误，避免 SSE 长时间悬挂。
- 修复前端“AI 正在处理您的消息...”卡住：SSE `reader` 结束时增加强制收尾（兜底 `markStreamComplete`），并将该提示仅限 `isStreaming=true` 时显示。
- 新增前端对 `chunk` 中 `{\"type\":\"result\",\"isError\":true}` 的错误识别：直接将错误内容渲染为消息文本（如 `Cannot read properties...`），并立即结束流式状态。
- 优化 `result/isError` 错误文案：将 `Cannot read properties of null (reading 'model')` 映射为可读提示（模型会话异常，建议重试或切换模型）。
- 修复错误消息“刷新后丢失”：`MessageList` 在回放历史 `stream-json` 时可解析 `result/isError` 并稳定展示错误文本。
- 修复 Codex 会话恢复异常：当 `resumeSession` 失败时自动降级为 `createSession` 重试；并允许对话会话 ID 映射更新，覆盖旧的失效会话 ID。
- 修复 SDK 成功判定：`runNeovateSdk` 识别 `result/isError`（或 `subtype=error`）并返回错误，避免将“业务错误结果”误判为可用。

## 影响

- 新建对话与会话消息发送时，可选择并透传 IFLOW 或 Codex GPT 模型。
- 既有 IFLOW 模型参数继续可用，不会因本次变更失效。
- 前后端模型枚举与文档保持一致，减少配置歧义。
- Codex OAuth 未登录或失效时，用户界面不再允许选择 `codex/*` 模型，避免无效请求。

## 验证

- 全局检索确认前后端模型常量同时包含 IFLOW 与 Codex GPT 系列。
- 检查 `spec-model-selection` 与 `spec-neovate-sdk-verify` 默认模型已同步为 `codex/gpt-5-codex`。
- 检查后端 `GET /api/conversations/models` 已返回 `enabled` 状态，并在前端下拉生效。
- 运行 `pnpm --dir backend test -- modelAvailabilityService.test.ts --runInBand`，验证 Codex 可用性测试通过。
- 运行 `pnpm --dir backend test -- conversationRoutes.performance.test.ts --runInBand`，验证路由改动后基础接口行为正常。

## 风险

- 依赖 Neovate/平台侧对所列 GPT 模型 ID 的可用性配置，若环境未开通将导致调用失败。
