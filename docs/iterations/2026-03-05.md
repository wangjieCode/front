## 补充（代码变更未展示修复）

- 问题：代码修改完成后，消息底部 `codeChanges` 与右侧 Review diff 为空。
- 根因：`NeovateAIService.parseOutput` 在 stream-json 未命中解析时，仅依赖当前 `git diff HEAD`，在特定执行路径下会拿不到变更，导致 `metadata.codeChanges=[]`。
- 修复：
  - 在 `modifyCode/modifyCodeStream` 执行前记录 Git 基线（`HEAD`）。
  - 当常规解析与当前 diff 均为空时，回退到“执行前后基线差异 + 工作区 diff”提取变更。
  - 文件：`backend/src/services/NeovateAIService.ts`
- 验证：
  - `pnpm -C backend build` 通过。

  - `WorktreeManager` 移除 `LruCacheService` + `CacheStrategyManager` 缓存读写，`getWorktreeInfo` 改为实时计算。
  - 前端与移动端移除模式选择 UI：
    - 删除 `frontend/src/components/ModeSelector.tsx`
    - 删除 `frontend/src/mobile-components/MobileModeSelector.tsx`
    - 同步精简 `App/AppMobile`、`ChatRoute/MobileChatRoute`、`ConversationView/MobileConversationView`、`MobileCreateConversation`、会话列表与相关类型/测试。
  - 同步更新规格与文档：
