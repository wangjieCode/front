# 2026-02-11

## 背景

- 新建对话输入区的图片上传按钮位置与继续对话输入区不一致，操作路径不统一。

## 变更

- 调整 `frontend/src/components/ConversationView.tsx` 新建对话分支底部操作区布局：
  - 将图片上传按钮从左侧提示区移动到右侧输入操作区（与模型选择、发送同区域）。
  - 将选图后的附件预览从输入框下方移动到输入框区域内部渲染。
  - 保留原有图片上传、附件预览、删除附件逻辑，不增加兼容分支。
- 新建对话区域视觉统一：
  - `ConversationView` 新建对话段落改为 class 驱动样式，移除大段内联样式。
  - 新增 `App.css` 中创建对话专用样式，统一卡片、表单、输入区、附件缩略图、模型选择与发送按钮的视觉规则。
  - 抽取 `handleSubmitCreateConversation`，合并 Enter 与按钮点击的重复提交逻辑。
  - 创建按钮文案从“发送”改为“创建对话”，加载态文案改为“创建中...”。
  - 对话模式分段控件改为等分铺满可用宽度，统一字体颜色与控件对齐。
  - 底部上传按钮/模型选择/创建按钮统一为同高 48px 对齐，操作区宽度约束一致。
  - 创建按钮移除“空输入置灰”逻辑，默认常亮；空内容点击时仍由提交校验给出提示。
  - 底部操作区移除固定宽度与最小宽度占位，改为按内容宽度收敛并右对齐，避免右侧区域视觉过宽。
  - 模型选择器宽度改为受控范围，创建按钮最小宽度收敛，减少操作区冗余横向占位。
  - 快捷键提示文案统一为“按 Ctrl/Cmd + Enter 创建对话”。
  - 修正底部模型选择器与创建按钮的垂直中线对齐，统一上传/模型/创建控件的视觉基线。
  - 下调底部上传按钮尺寸（44px）与模型选择器尺寸（宽度/高度/字体），压缩操作区视觉体量。
  - 下调附件缩略图删除按钮尺寸并改为缩略图内嵌定位，避免按钮越界后被容器截断。
  - 新建对话与继续对话输入框新增粘贴图片能力：识别剪贴板图片并追加到附件预览，走与本地上传一致的数据结构。
- 同步更新规格文档 `docs/specs/spec-conversations.md`，补充“新建对话与连续对话图片上传按钮位置一致”的用户体验要求。

## 影响

- 新建对话与继续对话的图片上传按钮位置一致，降低操作切换成本。
- 新建对话选图后，附件预览与输入内容处于同一区域，视觉关系更直接。
- 新建对话模块视觉风格更统一，信息层级更清晰，维护成本降低。
- 按钮语义更明确（创建而非发送），模式选择区域可读性与对齐一致性提升。
- 底部操作控件对齐一致，视觉重心稳定；创建按钮状态表达更直接。
- 底部操作区不再保留大块空白占位，横向布局更紧凑，和上方输入区视觉节奏更一致。
- 底部操作区内部控件对齐更稳定，模型下拉箭头与按钮文本不再出现视觉偏移。
- 图片选择与模型选择控件占位更小，底部操作区更轻量。
- 附件删除按钮体量与缩略图比例更协调，交互点位清晰且不再被裁剪。
- 无需先点上传按钮，截图可直接粘贴进输入框，附件添加路径更顺滑。
- 不影响图片附件数据结构、发送链路和后端接口。

## 本次追加（GitLab 分支查询排查日志）

### 背景

- `GET /api/conversations/gitlab/branches` 在 GitLab 查询失败时会回退为空分支与项目默认分支，缺少可直接定位问题来源的链路日志。

### 变更

- `backend/src/api/conversationRoutes.ts`
  - 新增接口入口、成功、失败日志，输出 `projectId`、`userId`、`branchesCount`、`defaultBranch`。
- `backend/src/services/ConversationManager.ts`
  - 新增分支查询参数与结果汇总日志，输出 `gitlabProjectId`、`branchesCount`、GitLab 默认分支、最终默认分支来源。
  - 新增空分支列表与默认分支回退告警日志。
- `backend/src/services/GitLabMCPService.ts`
  - `getProjectInfo` 新增请求开始、失败状态码、异常日志。
  - `listBranches` 新增分页结果、失败状态码、异常与完成总数日志。
  - GitLab 非 2xx 时追加失败响应体片段（截断到 300 字符）用于快速排查权限/配置问题。
- 同步更新规格文档 `docs/specs/baseline-branch.md`，补充基线分支接口可观测性要求。

### 影响

- 分支接口出现 `branches=[]` 或 `defaultBranch` 回退时，可通过日志快速区分为：
  - `gitlabProjectId` 不正确；
  - GitLab Token 权限/有效性问题；
  - GitLab 项目本身无分支；
  - GitLab API 请求异常。

### 验证

- 未运行自动化测试（本次为日志增强，未要求）。
- 代码检视确认三层日志已覆盖：路由入口、服务汇总、GitLab 请求细节。

### 风险

- 日志量增加（尤其分页分支列表场景）；已通过字段收敛与响应体截断控制体积。
