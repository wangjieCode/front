# 迭代记录：2026-01-31

## 背景

- 优化连续对话底部输入区视觉协调度与操作一致性。

## 变更

- 统一输入区容器、模型选择与发送按钮的高度与圆角风格。
- 收敛内联样式，补充聊天输入相关样式类。
- 连续对话底部输入区限制最大宽度并居中展示。
- 模型选择样式改为轻量胶囊下拉，减少突兀感。
- 模型选择折叠进更多菜单，输入区更简洁。
- 发送按钮强制圆形显示，确保与输入区比例一致。
- 发送按钮固定为 36x36，避免高度被按钮尺寸拉伸。
- AI 执行从 Neovate CLI 切换为 Neovate SDK。
- NeovateAIService 与 NeovateProvider 使用 SDK 输出流，移除命令行构造逻辑。
- 移除 NEOVATE_PATH/NEOVATE_ARGS 配置，补充 IFLOW_API_KEY 说明。
- 更新 AI 执行与模型选择相关文档为 SDK 描述。
- 移除 RUN_MODE=remote 模式与 SSH 执行逻辑，统一使用本地执行器。
- Git/Worktree 配置收敛为 LOCAL_GIT_WORK_DIR + WORKTREE_BASE_DIR，删除远程与本地分支配置。
- 前端 SSE 流式处理兼容 Neovate SDK `result` 事件，确保无 `complete` 时也能结束状态与拉取元数据。
- Neovate SDK 接入改为 `createSession/resumeSession` 会话模式，按 SDK 消息流输出 JSON 行。
- 会话 ID 使用 SDK session 对象返回的 `sessionId`，不再依赖解析输出行。
- 新增对话中断接口与前端停止按钮，支持终止流式响应。
- 中断后停止流式状态并将用户消息回填到输入框。

## 影响

- 连续对话页底部输入区观感更一致，操作区对齐更清晰。
- AI 执行依赖 Neovate SDK 包，后端运行环境需可访问 SDK。
- 初始化与脚本执行仅支持本地目录与本地执行器。

## 风险

- 自定义样式覆盖 Ant Design 默认样式，若主题色调整可能需要同步微调。
- SDK 输出格式或运行环境变更可能影响解析结果，需要关注 SDK 版本升级。
- 运行环境必须显式配置 LOCAL_GIT_WORK_DIR 与 WORKTREE_BASE_DIR，否则启动失败。
