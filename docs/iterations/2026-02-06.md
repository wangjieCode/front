# 迭代记录：2026-02-06

## 背景

- 前端服务层保留了两个后端未实现接口的遗留调用定义，增加维护噪音。

## 变更

- 删除前端 `conversationService` 中 `getSessionStatus` 与 `getMessage` 方法。
- 删除前端 `conversationService` 中基于 `/api/conversations/:sessionId/status` 的整套轮询逻辑与相关状态字段。
- 同步更新接口文档与会话规格，移除对应已知缺口描述。
- 删除前端 `ConversationView` 中基于 Neovate `result` 事件的流结束兼容逻辑，仅保留 SSE `complete` 结束事件。
- 删除前端 `ConversationView` 中 `fallbackBranch` 分支兜底逻辑，分支加载失败时直接清空选项与值。
- 删除前端 `neovateParser` 未使用的纯文本兼容导出（`parseNeovateStreamJson`、`parseNeovateChunk`），统一保留结构化解析入口。
- 移动端首页将“页面标题区”和“导航 Tabs 区”融合为单一卡片容器，减少重复卡片层级与留白。
- 移动端 Chat 页输入区改为底部悬浮固定，消息区增加底部留白以避免内容遮挡。
- 移动端全局头部与页面头部区域改为顶部吸顶，滚动内容时保持头部可见。
- 调整移动端头部吸顶策略：仅保留全局 Header 固定，页面级头部取消吸顶以避免与内容重叠。
- 移动端首页移除标题与描述区，将“新对话”改为右下角悬浮按钮，释放首屏垂直空间。
- 移动端“新对话”悬浮按钮改为全页面显示，点击统一跳转到新建页；在 Chat 页自动上移避免遮挡输入区。
- 移动端“新对话”悬浮按钮位置调整为右上角固定（位于全局 Header 下方）。
- 移动端“新对话”入口从悬浮按钮改为 Tabs 导航栏右侧按钮，统一导航区操作入口。
- 移动端 `mobile-page-shell`（Tabs 导航区）上移到顶部 Header 区域，实现头部导航一体化。
- 移动端顶部区域布局收敛：压缩头部高度、统一 Tabs 行与右侧按钮对齐，并增加窄屏防挤压样式。
- 移动端首页视觉细化：顶部与容器增加圆角和毛玻璃效果，统一边距与阴影层级。
- 修正移动端顶部布局观感：将 Header 调整为独立圆角玻璃卡片，优化 Tabs 与“新对话”按钮对齐关系。
- 修复移动端 Header 裁切回归：覆盖 Ant Header 默认固定高度，恢复 Tabs 区完整显示并同步调整内容顶部偏移。
- 移动端头部两行独立化：第一行品牌区与第二行导航区拆分为独立卡片，分层更明确。
- 移动端视觉主题回退为白色底卡片，移除毛玻璃效果，保持轻边框与阴影。
- 重构移动端抽屉“对话列表”样式：改为卡片化列表、优化标题/元信息层级、统一模式与删除操作区。
- 优化移动端对话卡片内部对齐：统一首行标签与次行时间/删除按钮的基线与间距。
- 统一“私密/编辑”标签规格（高度、字号、内边距）以修复顶部标签未对齐问题。
- 修复生产入口分流短路：静态资源中间件禁用默认 `index.html` 直出，确保 `/` 请求进入 PC/Mobile 分流逻辑。
- 同步开发环境分流策略：Vite dev server 增加与生产一致的 UA 重定向规则（`/` <-> `/m`）。
- 为生产与开发分流补充 `X-Entry-Route` 响应头，便于用 `curl -I` 快速定位命中路径。
- 删除后端 `/api/docker/*` 与 `/api/docker-compose/*` 路由注册，移除对外 Docker 运维 API 暴露面。
- 删除后端 `dockerRoutes`、`dockerComposeRoutes`、`DockerService`、`DockerComposeService` 文件与对应实现。
- 删除后端 `package.json` 中 Docker 相关脚本与 `dockerode`/`ssh2` 依赖声明。
- 删除 Docker 管理相关规格与技术文档（`spec-docker-admin`、`spec-docker-compose`、`docker_admin_api`），并同步更新架构与 API 参考文档。
- 为预览接口 `POST/GET/DELETE /api/conversations/:sessionId/preview*` 接入 `requireAuth`，收敛未鉴权入口。

## 影响

- 前端不再维护未落地接口的调用路径，服务层逻辑更精简。
- 现有对话主流程（创建、消息流式、归档、MR、预览）不受影响。
- 前端流式结束判定路径收敛，分支选择状态管理更简单。
- 解析器减少重复逻辑与无调用导出，维护成本下降。
- 移动端首页视觉层级更简化，首屏信息块更聚合。
- 移动端对话页发送入口可持续可见，长会话滚动时发送操作更稳定。
- 移动端首屏导航与页面上下文在滚动过程中保持稳定，降低定位成本。
- 消除页面级头部与会话内容的重叠问题，滚动阅读连续性更好。
- 移动端首页首屏信息密度进一步降低，关键入口更聚焦。
- 新建入口跨页面一致可达，页面切换成本降低。
- 新建入口与阅读区、输入区分离，减少底部操作区域冲突。
- 顶部操作入口更聚合，页面浮层元素减少，视觉更稳定。
- 页面内容区首屏高度提升，减少重复顶部块占用。
- 顶部导航在不同屏宽下可读性更稳定，减少元素拥挤与换行风险。
- 页面层级更清晰，卡片边界更柔和，整体观感更轻。
- 顶部信息区边界与分层更明确，导航与操作按钮视觉冲突降低。
- 消除顶部导航截断与空白带问题，页面首屏结构恢复稳定。
- 头部结构语义更清晰，视觉节奏更稳定。
- 视觉风格更稳定朴素，减少半透明叠层带来的噪声。
- 对话列表可读性与点击区域清晰度提升，抽屉观感更统一。
- 对话项信息布局更规整，降低“时间与操作按钮错位”观感。
- 顶部状态标签视觉基线一致性提升。
- 生产环境移动端 User-Agent 访问 `/` 可正确重定向到 `/m`。
- 开发环境与生产环境入口行为一致，降低联调偏差。
- 分流问题排查可观测性提升，降低定位成本。
- 后端对外接口面收敛，避免通过 API 触发 Docker 运维操作。
- 代码结构与文档一致性提升，减少已下线能力的维护噪音与误用风险。
- 预览相关操作需登录后才能访问，降低匿名调用风险。
