# 迭代记录：2026-03-06

## 背景

- 需要将 task 队列与业务缓存的 Redis 使用彻底隔离，避免同实例互相影响。
- 需要删除项目中的进程内 LRU 内存缓存，统一替换为 Redis 缓存实现，简化缓存链路。

## 变更

- 后端缓存实现替换：
  - 新增 `backend/src/services/RedisCacheService.ts`，统一提供 `getJson/setJson/del/delByPattern`。
  - 删除 `backend/src/services/LruCacheService.ts`。
  - `CacheStrategyManager` 改为依赖通用缓存接口 `CacheClient`，不再绑定 LRU 实现。
  - `ConversationManager`、`ProjectService`、`BranchCacheService` 缓存实现切换为 `RedisCacheService`。
- Redis 配置隔离：
  - `QueueManager` 改为仅读取 `TASK_REDIS_URL`、`TASK_REDIS_PREFIX`。
  - 业务缓存改为仅读取 `BIZ_REDIS_URL`、`BIZ_REDIS_PREFIX`。
- 依赖与测试调整：
  - `backend/package.json` 删除 `lru-cache`，新增 `ioredis`。
  - 删除 `backend/src/__tests__/lruCacheService.test.ts`。
  - `backend/src/__tests__/cacheStrategyManager.test.ts` 改为使用内存 `CacheClient` stub，移除对 LRU 的依赖。
- 文档同步：
  - 更新环境变量、服务职责、数据模型与会话/项目规格文档，删除“业务缓存为 LRU”描述并补充 task/biz Redis 隔离约束。

## 影响

- 业务缓存从进程内内存转为 Redis，缓存可跨进程共享，不再依赖单进程状态。
- task 队列与业务缓存使用不同 Redis 配置，降低队列流量/抖动对业务缓存的影响。
- 原有 LRU 持久化相关能力（落盘快照）已整体移除，不再保留兼容逻辑。

## 验证

- 执行 `pnpm -C backend test -- cacheStrategyManager.test.ts --runInBand`，通过。
- 执行 `pnpm -C backend build`，通过。

## 风险

- 若部署环境未补齐 `TASK_REDIS_URL` 或 `BIZ_REDIS_URL`，对应模块会在启动或首次使用时抛错。
- 业务缓存当前默认无 TTL，需继续依赖写后删键保证一致性，后续如新增缓存点需严格遵循该约束。

## 补充（LOCAL_GIT_WORK_DIR 专属 Git 配置 + GITLAB_TOKEN Bearer 复用）

### 背景

- 需要将 Git 运行配置限制在远程 `LOCAL_GIT_WORK_DIR` 生效，避免污染全局 `~/.gitconfig`。
- 需要移除 URL 内嵌 token 方案，统一改为 Bearer 认证并仅在网络命令注入。

### 变更

- 执行器认证注入收敛：
  - `backend/src/services/LocalExecutor.ts`
    - 新增 Git 网络命令识别（`fetch/pull/push/clone/ls-remote/remote update`）。
    - 命中网络命令且存在 `GITLAB_TOKEN` 时，自动注入 `GIT_HTTP_EXTRAHEADER=Authorization: Bearer <token>`。
    - 本地命令不注入认证头。
- 仓库服务移除 URL token 改写：
  - `backend/src/services/RepositoryService.ts`
    - 删除 `withGitlabToken()`、`ensureAuthRemote()`、`getAuthUrl()`。
    - `ls-remote/clone/pull` 路径统一使用原始 URL，认证由执行器注入处理。
  - `backend/src/services/ProjectService.ts`
    - 删除 `git remote set-url origin <token-url>` 逻辑。
- 部署链路对齐：
  - 新增 `backend/.gitconfig.workspace`（工作区最小 Git 配置）。
  - `deploy.mjs`
    - 将 `backend/.gitconfig.workspace` 作为发布必需文件。
    - 发布时同步到远程 `${DEPLOY_DIR}/.gitconfig.workspace`。
    - 自动写入/更新 `backend/.env` 的 `GIT_CONFIG_GLOBAL=${DEPLOY_DIR}/.gitconfig.workspace`。
    - 部署阶段执行 `git config --show-origin --list` 抽样校验配置来源包含 `.gitconfig.workspace`。
- 文档同步：
  - 更新 `docs/specs/spec-projects.md`。
  - 更新 `docs/technical/worktree_and_git.md`。

### 验证

- 执行 `pnpm -C backend test -- src/services/__tests__/LocalExecutor.test.ts --runInBand`，通过（5/5）。
- 执行 `pnpm -C backend build`，通过。

### 风险

- 若目标 Git 服务器不支持 `Authorization: Bearer`，网络命令会鉴权失败（本次按既定策略不做 Basic/oAuth2 回退）。
- 若远程启动脚本未读取 `backend/.env`，需确保启动链路仍导出 `GIT_CONFIG_GLOBAL`。
