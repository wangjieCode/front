# 规格：Merge Request

## 基本信息

- 名称：MR 创建
- 负责人：未指定
- 创建日期：2026-01-30
- 最近更新：2026-03-05

## 背景

- 需要将会话分支变更进入审查流程。

## 目标

- 支持会话内手动创建 MR。

## 非目标

- 不提供自动合并。

## 范围

- In：创建 MR、保存 MR URL。
- Out：审批与合并策略。

## 业务规则

- 仅会话创建者允许创建 MR。

## 需求

### 功能需求

- F1：调用 GitLab API 创建或查询 MR。
- F2：保存 MR URL 到上下文。

### 非功能需求

- N1：失败需返回明确错误。

## 用户体验

- 已存在 MR 时直接返回链接。

## 数据与接口

- conversation_contexts.mr_url。
- API：POST /api/conversations/:sessionId/merge-request。

## 验收标准

- A1：创建成功返回 MR URL。
- A2：缺少会话分支信息时返回错误。

## 风险与依赖

- 依赖 GitLab 配置与权限。
