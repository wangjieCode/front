# 规格：会话与消息

## 基本信息

- 名称：会话与消息
- 负责人：未指定
- 创建日期：2026-01-30
- 最近更新：2026-01-30

## 背景

- 需要以会话为单位组织对话与代码修改。

## 目标

- 支持创建会话、发送消息、流式响应。
- 通过状态控制写操作。

## 非目标

- 不支持会话恢复为 ACTIVE（归档不可逆）。

## 范围

- In：会话创建、消息发送、归档、可见性。
- Out：复杂多状态机。

## 业务规则

- 会话状态只有 ACTIVE / ARCHIVED。
- 归档后禁止发送消息与创建 MR/预览。
- 可见性为 PRIVATE / PUBLIC。

## 需求

### 功能需求

- F1：创建会话并保存上下文。
- F2：消息通过 SSE 返回。
- F3：会话可归档。
- F4：可更新会话可见性。

### 非功能需求

- N1：消息响应需支持流式输出。

## 用户体验

- 发送消息立即看到用户输入与“思考中”。
- 流式返回增量展示。
- 创建对话选择项目后不展示仓库与工作目录。
- 连续对话底部输入区与模型切换、发送按钮保持同一高度与视觉风格。

## 数据与接口

- conversations / conversation_contexts / messages / message_metadata。
- API：/api/conversations 及子路由。

## 验收标准

- A1：ACTIVE 会话可发送消息。
- A2：ARCHIVED 会话发送消息返回 403。
- A3：可见性切换成功。
