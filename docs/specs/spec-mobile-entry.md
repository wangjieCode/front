# 移动端入口与分流

## 目标

- 移动端使用独立入口构建与部署，避免与桌面端互相干扰。
- 后端通过 User-Agent 判断并分流到移动端或桌面端入口。
- 移动端与桌面端复用相同的业务逻辑与共享组件。

## 范围

- 仅包含入口选择与路由分流策略。
- 不涉及 UI 细节与具体页面样式。

## 设计

### 前端入口

- 桌面入口：`index.html` -> `src/main.tsx` -> `App`。
- 移动入口：`mobile.html` -> `src/mobile.tsx` -> `AppMobile`。
- 移动端路由基准为 `/m`，确保所有内部导航一致。

### 后端分流

- 新增移动端入口路由 `/m`。
- 对非 API 请求，根据 User-Agent 判断是否为移动端：
  - 若为移动端：重定向到 `/m` 前缀路径（保留原路径与查询参数）。
  - 若为桌面端：返回桌面入口 `index.html`。
- 若请求以 `/m` 开头且为桌面端，则重定向到对应的桌面路径。
- `/m` 与其子路径始终返回 `mobile.html`（除非命中静态资源）。

### 开发环境

- 本地 Vite 开发服务为 `/m` 与其子路径返回 `mobile.html`，确保开发与生产路由一致。

## 约束

- 不做多入口兼容回退；入口选择以 `/m` 与 User-Agent 结果为准。
- 入口分流只作用于非 `/api` 请求。
- 移动端与桌面端 UI 组件完全分离，不共享 UI 组件与样式。
- 仅复用纯逻辑模块（如消息解析、接口调用、数据处理等）。
- 移动端介绍页使用移动优先版式，避免直接套用桌面布局。

## 验证

- 不要求本地启动验证，按迭代记录说明。
