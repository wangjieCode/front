# Neovate SDK 验证脚本规格

## 背景

- 现状：Neovate SDK 仅依赖 `IFLOW_API_KEY` 环境变量，缺少可直接验证可用性的脚本。
- 问题：无法快速确认 API Key 是否可用、模型是否可正常交互。

## 目标

- 提供一个本地脚本验证 `IFLOW_API_KEY` 是否可用。
- 验证能与指定模型完成一次最小交互。

## 非目标

- 不做复杂对话或多轮验证。
- 不验证仓库初始化、文件变更或提交流程。
- 不替代后端服务健康检查。

## 范围

- In：后端 `scripts/` 下提供独立验证脚本。
- Out：前端 UI 或服务端 API 端点。

## 业务规则

- 必须显式检查 `IFLOW_API_KEY` 是否存在。
- 日志只展示脱敏后的 API Key（保留前后 4 位）。
- 优先加载 `.env.production`，不存在时再加载 `.env`。
- 默认使用 `iflow/qwen3-coder-plus` 模型，允许命令行覆盖。
- 在超时时间内未收到响应视为失败。

## 需求

### 功能需求

- F1：脚本读取 `IFLOW_API_KEY`，为空则直接失败退出。
- F2：脚本使用 Neovate SDK `createSession` 发送一次提示词并接收响应。
- F3：收到成功结果或助手消息即判定验证成功，否则失败。
- F4：支持 `--model`、`--workdir`、`--prompt`、`--timeout` 参数。

### 非功能需求

- N1：输出精简可读的验证日志。
- N2：失败时返回非 0 退出码。

## 用户体验

- 关键流程：`pnpm --dir backend verify:neovate [options]` → 使用 Node ESM 脚本运行 SDK → 输出模型与工作目录 → 返回成功/失败。
- 失败提示：明确提示 API Key 缺失、超时或模型返回错误。

## 数据与接口

- 数据结构：不新增。
- 接口清单：不新增。

## 验收标准

- A1：`IFLOW_API_KEY` 缺失时，脚本立即失败并提示配置环境变量。
- A2：API Key 可用时，在超时时间内收到模型响应并返回成功。
- A3：支持通过参数覆盖默认模型与提示词。

## 风险与依赖

- 风险：SDK 升级导致消息结构变化时需同步调整脚本解析逻辑。
- 依赖：运行环境可访问 Neovate SDK 与网络。
