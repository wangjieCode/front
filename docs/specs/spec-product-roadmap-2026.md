# 规格：产品命名与 2026 迭代规划

## 基本信息

- 名称：代码伙计（Code Mate）命名与 2026 Roadmap
- 负责人：产品与研发协作组
- 创建日期：2026-02-24
- 最近更新：2026-02-24

## 背景

- 现状：当前产品具备“对话驱动改码 + 会话隔离 + 交付链路”基础能力，但缺少全年统一节奏与命名锚点。
- 问题：需求优先级容易按临时问题驱动，跨端体验、稳定性与可观测性投入节奏不稳定。

## 目标

- 目标 1：统一对外产品名称为 `代码伙计 / Code Mate`。
- 目标 1.1：统一对外 slogan 为 `少折腾，多交付。/ Less hassle, more delivery.`。
- 目标 2：形成 2026 年可执行迭代主线，按季度推进。
- 目标 3：将迭代重点从“功能堆叠”切到“效率、稳定性、可运营”。

## 非目标

- 不做：2026 年内不新增完全独立的第二产品线。
- 不做：不保留旧命名并行对外传播。

## 范围

- In：文档命名统一、年度主题、季度重点、核心衡量指标。
- Out：具体周排期与人力预算拆分。

## 业务规则

- 规则 1：对外文档、业务文档统一使用 `代码伙计 / Code Mate`。
- 规则 1.1：官网/应用页头、页面标题与介绍页统一使用 slogan `少折腾，多交付。/ Less hassle, more delivery.`。
- 规则 2：季度迭代优先级遵循“稳定性 > 核心效率 > 新能力”。
- 规则 3：每季度至少落地 1 项可量化效率指标改进。

## 需求

### 功能需求

- F1：2026 Q1（基础稳定）
  - 会话、项目、消息链路性能基线固化，补全慢请求定位闭环。
  - 完成移动端与桌面端核心流程一致性收敛（创建、发送、查看轨迹）。
- F2：2026 Q2（交付提效）
  - 增强变更摘要与风险提示质量，减少人工二次审查时间。
  - 打通“改动 -> 预览 -> 提交 -> MR”的最短操作路径。
- F3：2026 Q3（团队协作）
  - 增加会话级协作能力（评论、@提醒、关键节点标记）。
  - 建立多项目视角的迭代看板与会话状态追踪。
- F4：2026 Q4（规模化运营）
  - 建立质量评分面板（成功率、回滚率、交付时长）。
  - 完成核心接口限流与故障演练机制，支撑更高并发。

### 非功能需求

- N1：核心 API P95 响应时间季度环比下降。
- N2：关键链路失败率季度环比下降。
- N3：文档与规格更新覆盖率保持 100%（有代码变更即有规格/迭代记录）。

## 用户体验

- 关键流程：创建对话后，用户能在单一流程内完成需求输入、变更查看、预览验证、提交交付。
- 失败提示：所有关键失败场景提供可执行下一步建议（重试、回滚、人工接管）。

## 数据与接口

- 数据结构：
  - `roadmap_quarter`：季度编号、目标、KPI、状态。
  - `delivery_metrics`：交付时长、失败率、回滚率。
- 接口清单：
  - 沿用现有项目/会话/消息/预览/MR 接口；2026 年以链路优化与观测增强为主。

## 验收标准

- A1：全量文档中产品名称统一为 `代码伙计 / Code Mate`。
- A1.1：主要对外入口（桌面/移动介绍页、应用标题）统一展示 slogan `少折腾，多交付。/ Less hassle, more delivery.`。
- A2：形成可读的 2026 Q1-Q4 迭代主线，并能映射到后续迭代记录。
- A3：季度复盘可直接按本规格中的 KPI 维度评估完成度。

## 风险与依赖

- 风险：过多并行主题会稀释执行力，导致季度目标不收敛。
- 依赖：前后端、产品、运维需要统一季度节奏与验收口径。
