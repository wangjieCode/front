# Query 任务执行修复需求文档

## 简介

修复 query 类型任务的执行问题，确保 AI 能够正确处理只读查询请求并返回有意义的结果。

## 术语表

- **Query Task**: 只读模式的任务，不修改代码，仅查询和分析代码库
- **Code Change Task**: 编辑模式的任务，会修改代码并创建 MR
- **CodeTool**: AI 代码助手工具（如 neovate 或 qoder-cli）
- **Task Orchestrator**: 任务编排器，负责协调任务执行流程

## 需求

### 需求 1: Query 任务应调用 AI 工具

**用户故事**: 作为用户，当我发送一个 query 类型的任务时，我希望 AI 能够分析我的问题并返回有意义的答案，而不是简单地返回我的问题本身。

#### 验收标准

1. WHEN 用户创建 query 类型任务 THEN 系统应调用 AI 代码工具处理该查询
2. WHEN AI 工具处理查询 THEN 系统应以只读模式运行，不修改任何代码
3. WHEN AI 工具返回结果 THEN 系统应将结果保存到任务的 result 字段
4. WHEN 任务执行完成 THEN 系统应将任务状态更新为 SUCCESS
5. WHEN 任务执行失败 THEN 系统应将任务状态更新为 FAILED 并记录错误信息

### 需求 2: 区分只读和编辑模式

**用户故事**: 作为系统开发者，我希望能够明确区分只读模式和编辑模式的执行流程，以便正确处理不同类型的任务。

#### 验收标准

1. WHEN 任务类型为 query THEN 系统应以只读模式调用 AI 工具
2. WHEN 任务类型为 code_change THEN 系统应以编辑模式调用 AI 工具
3. WHEN 只读模式执行 THEN 系统应跳过分支创建、代码提交和 MR 创建步骤
4. WHEN 编辑模式执行且有代码变更 THEN 系统应执行完整的 Git 工作流
5. WHEN 编辑模式执行但无代码变更 THEN 系统应将 AI 的输出作为查询结果返回

### 需求 3: 流式输出支持

**用户故事**: 作为用户，我希望在 AI 处理查询时能够实时看到输出内容，而不是等待所有内容完成后才显示。

#### 验收标准

1. WHEN AI 工具开始处理查询 THEN 系统应通过 WebSocket 实时发送输出内容
2. WHEN AI 工具输出新内容 THEN 前端应立即显示该内容
3. WHEN AI 工具完成处理 THEN 系统应发送完成信号
4. WHEN 发生错误 THEN 系统应通过 WebSocket 发送错误信息
5. WHEN 流式输出结束 THEN 系统应将完整输出保存到任务结果中

### 需求 4: 前端结果展示

**用户故事**: 作为用户，我希望在前端能够清晰地看到 AI 对我查询的回答，包括思考过程和最终结果。

#### 验收标准

1. WHEN query 任务完成 THEN 前端应在 AI 响应区域显示结果
2. WHEN 结果包含 JSON 格式的流式输出 THEN 前端应解析并提取最终结果
3. WHEN 结果是纯文本 THEN 前端应直接显示文本内容
4. WHEN 任务失败 THEN 前端应显示错误信息
5. WHEN 任务执行中 THEN 前端应显示加载状态和实时输出

### 需求 5: 日志记录

**用户故事**: 作为系统管理员，我希望能够查看 query 任务的详细执行日志，以便调试和监控系统运行状态。

#### 验收标准

1. WHEN query 任务开始执行 THEN 系统应记录任务开始日志
2. WHEN 调用 AI 工具 THEN 系统应记录工具调用日志和提示词
3. WHEN AI 工具返回输出 THEN 系统应记录输出内容
4. WHEN 任务完成 THEN 系统应记录任务完成日志和执行时长
5. WHEN 发生错误 THEN 系统应记录详细的错误堆栈信息

### 需求 6: 错误处理

**用户故事**: 作为用户，当 query 任务执行失败时，我希望能够看到清晰的错误信息，以便了解失败原因。

#### 验收标准

1. WHEN AI 工具不可用 THEN 系统应返回工具不可用的错误信息
2. WHEN AI 工具执行超时 THEN 系统应返回超时错误信息
3. WHEN AI 工具返回错误 THEN 系统应将错误信息传递给用户
4. WHEN 网络连接失败 THEN 系统应返回连接失败的错误信息
5. WHEN 发生未知错误 THEN 系统应记录完整错误堆栈并返回友好的错误提示

### 需求 7: 兼容性

**用户故事**: 作为系统开发者，我希望修复后的实现能够兼容现有的代码变更任务流程，不影响其他功能。

#### 验收标准

1. WHEN 执行 code_change 任务 THEN 系统应按原有流程执行
2. WHEN code_change 任务有代码变更 THEN 系统应创建分支、提交代码和创建 MR
3. WHEN code_change 任务无代码变更 THEN 系统应将 AI 输出作为结果返回
4. WHEN 切换任务类型 THEN 前端应正确显示不同类型任务的结果
5. WHEN 查看历史任务 THEN 系统应正确显示所有类型任务的执行记录
