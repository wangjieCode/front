# Web 前端实习生助手系统 - 需求文档

## 项目概述

一个基于 Web 的智能前端开发助手系统，通过可视化界面接收用户指令，在远程虚拟机上执行代码操作，并将结果反馈到 Web 端展示。系统集成 qodercli 进行代码修改，并通过 GitLab MCP 自动创建 Merge Request。

**核心价值：** 虚拟机提供完整的前端开发环境（Node.js、npm、构建工具等），让产品经理、后端工程师等非前端专业人员无需配置本地环境，即可通过自然语言描述需求，专注于业务逻辑和代码修改，降低前端开发门槛。

## 用户故事

### US-1: Web 界面交互（零门槛操作）
**作为** 产品经理/后端工程师/非前端人员  
**我想要** 通过 Web 界面用自然语言描述需求，无需了解前端技术细节  
**以便于** 专注于业务逻辑和功能实现，而不是环境配置和命令行操作

**验收标准：**
- AC-1.1: Web 界面提供输入框，用户可以输入自然语言提示词（例如："修改登录按钮颜色为蓝色"、"添加用户列表分页功能"）
- AC-1.2: 提交后显示加载状态，提示系统正在处理
- AC-1.3: 执行结果以可视化方式展示（包括成功/失败状态、日志输出、代码变更等）
- AC-1.4: 支持历史记录查看，可以回溯之前的操作
- AC-1.5: 界面响应式设计，支持桌面和移动端访问
- AC-1.6: 提供常用操作模板（如"运行项目"、"执行测试"、"修复 lint 错误"）
- AC-1.7: 智能提示和引导，帮助用户更好地描述需求

### US-2: 远程虚拟机管理（完整前端开发环境）
**作为** 产品经理/后端工程师/非前端人员  
**我想要** 使用预配置好的远程虚拟机开发环境  
**以便于** 无需在本地安装 Node.js、npm 等工具，直接进行前端开发

**验收标准：**
- AC-2.1: 支持通过 SSH 连接到远程虚拟机
- AC-2.2: 虚拟机预装完整前端开发环境：
  - Node.js（LTS 版本）
  - npm/yarn/pnpm 包管理器
  - 常用构建工具（webpack, vite, rollup 等）
  - Git 版本控制
  - qodercli AI 代码工具
- AC-2.3: 能够在虚拟机上执行 shell 命令
- AC-2.4: 实时捕获命令执行的标准输出和错误输出
- AC-2.5: 支持虚拟机连接状态监控（在线/离线）
- AC-2.6: 提供虚拟机配置管理（IP、端口、认证信息等）
- AC-2.7: 虚拟机环境版本可配置和更新（Node 版本、工具版本等）

### US-3: 代码仓库管理
**作为** 用户  
**我想要** 在虚拟机上管理 Git 代码仓库  
**以便于** 进行版本控制和代码协作

**验收标准：**
- AC-3.1: 支持克隆 Git 仓库到虚拟机
- AC-3.2: 支持切换分支、创建新分支
- AC-3.3: 支持查看当前仓库状态（修改文件、当前分支等）
- AC-3.4: 支持提交代码变更（git add, commit）
- AC-3.5: 支持推送代码到远程仓库

### US-4: AI 代码修改（qodercli）
**作为** 用户  
**我想要** 使用 AI 自动修改代码  
**以便于** 快速实现功能需求或修复问题

**验收标准：**
- AC-4.1: 在虚拟机上安装并配置 qodercli
- AC-4.2: 将用户提示词转换为 qodercli 命令
- AC-4.3: 执行 qodercli 并捕获其输出
- AC-4.4: 解析 qodercli 的代码变更结果
- AC-4.5: 将代码变更详情返回给 Web 端展示

### US-5: 代码执行与测试
**作为** 用户  
**我想要** 在虚拟机上运行代码和测试  
**以便于** 验证代码功能是否正常

**验收标准：**
- AC-5.1: 支持执行前端项目的构建命令（npm install, npm run build 等）
- AC-5.2: 支持运行测试命令（npm test, npm run lint 等）
- AC-5.3: 捕获并解析构建/测试输出
- AC-5.4: 识别错误和警告信息
- AC-5.5: 将执行结果格式化后展示在 Web 端

### US-6: Web 端可视化展示
**作为** 用户  
**我想要** 在 Web 界面上清晰地看到所有操作结果  
**以便于** 理解系统执行了什么以及结果如何

**验收标准：**
- AC-6.1: 展示命令执行日志（带语法高亮）
- AC-6.2: 展示代码变更对比（diff 视图）
- AC-6.3: 展示文件树结构变化
- AC-6.4: 展示构建/测试结果（成功率、错误信息等）
- AC-6.5: 支持展开/折叠详细信息
- AC-6.6: 提供下载日志或结果的功能

### US-7: GitLab MCP 集成
**作为** 用户  
**我想要** 标记任务完成后自动创建 Merge Request  
**以便于** 简化代码审查流程

**验收标准：**
- AC-7.1: 集成 GitLab MCP 工具
- AC-7.2: 用户可以在 Web 界面标记任务为"完成"
- AC-7.3: 标记完成后，系统自动推送代码到新分支
- AC-7.4: 自动创建 Merge Request，包含任务描述和代码变更摘要
- AC-7.5: 在 Web 界面显示 MR 链接和状态
- AC-7.6: 支持配置 MR 的目标分支、审查者等信息

### US-8: 任务管理
**作为** 用户  
**我想要** 管理多个开发任务  
**以便于** 组织和跟踪工作进度

**验收标准：**
- AC-8.1: 支持创建新任务，包含任务描述和需求
- AC-8.2: 每个任务关联一个独立的工作分支
- AC-8.3: 显示任务列表，包含状态（进行中/已完成/失败）
- AC-8.4: 支持切换不同任务进行操作
- AC-8.5: 任务完成后保留历史记录和操作日志

## 非功能性需求

### NFR-1: 性能
- 系统响应时间：Web 界面操作响应 < 200ms
- 命令执行反馈：实时流式输出，延迟 < 1s
- 支持并发：至少支持 10 个用户同时使用

### NFR-2: 安全性
- 虚拟机连接使用 SSH 密钥认证
- 敏感信息（密钥、token）加密存储
- 用户操作需要身份验证
- 限制可执行的命令范围，防止恶意操作

### NFR-3: 可靠性
- 虚拟机连接断开后自动重连
- 长时间运行的命令支持超时控制
- 操作失败时提供明确的错误信息和恢复建议

### NFR-4: 可维护性
- 代码模块化，前后端分离
- 提供详细的日志记录
- 配置文件化，便于部署和调整

### NFR-5: 可扩展性
- 支持添加更多 AI 工具（不限于 qodercli）
- 支持多种代码托管平台（不限于 GitLab）
- 支持多虚拟机管理

## 技术约束

- Web 前端：现代浏览器支持（Chrome, Firefox, Safari, Edge）
- 虚拟机：Linux 系统，支持 SSH 连接
- qodercli：需要在虚拟机上安装并配置
- GitLab MCP：需要 GitLab API 访问权限
- 网络：Web 服务器与虚拟机之间需要网络连通

## 目标用户

### 主要用户群体
1. **产品经理**: 需要快速验证原型、调整 UI 细节，但不熟悉前端技术栈
2. **后端工程师**: 需要修改前端接口调用、数据展示逻辑，但不想配置前端环境
3. **测试工程师**: 需要修复简单的前端 bug，但缺乏前端开发经验
4. **设计师**: 需要调整样式和交互，但不熟悉代码编写

### 使用场景
- 快速修改 UI 文案、颜色、布局
- 调整数据展示逻辑和格式
- 修复简单的前端 bug
- 添加或修改表单验证规则
- 调整 API 调用参数和响应处理
- 运行和查看测试结果

## 术语表

- **qodercli**: AI 驱动的代码编辑命令行工具
- **MCP**: Model Context Protocol，用于集成外部服务的协议
- **MR**: Merge Request，GitLab 中的代码合并请求
- **虚拟机**: 远程 Linux 主机，预装完整前端开发环境，用于执行代码操作
- **提示词**: 用户输入的自然语言指令，描述想要实现的功能或修改
- **前端开发环境**: 包含 Node.js、npm、构建工具、Git 等的完整工具链
